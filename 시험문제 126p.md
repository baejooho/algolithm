# 2차원 배열 원소 위치 계산 — 시험 완벽 대비

> **슬라이드 13번 / 교재 126p 기반**  
> 어떤 유형이든 이 문서 하나로 풀 수 있도록 정리

---

## 1. 핵심 개념 먼저 잡기

### 왜 위치를 계산해야 하는가?

2차원 배열은 **논리적으로는 표(행×열)** 형태지만,  
메모리(물리적)는 **1차원으로 연속 저장**됨.

```
논리적 구조 (2×4 배열)        물리적 메모리 구조
┌─────┬─────┬─────┬─────┐    [0][0][1][2][3] → [1][0][1][2][3]
│[0][0]│[0][1]│[0][2]│[0][3]│    주소 100 101 102 103   104 105 106 107
├─────┼─────┼─────┼─────┤    (int 배열, 각 원소 4byte)
│[1][0]│[1][1]│[1][2]│[1][3]│
└─────┴─────┴─────┴─────┘
```

→ **행 우선 순서(Row Major Order)**: C 언어가 사용하는 방식  
→ 0행 전체 → 1행 전체 → 2행 전체 순으로 저장

---

## 2. 공식 (무조건 암기)

### 행 우선 순서 (C 언어 사용)

$$\boxed{A[i][j]의\ 위치 = \alpha + (i \times n_j + j) \times \ell}$$

| 기호 | 의미 | 혼동 주의 |
|------|------|----------|
| α (alpha) | 배열 시작 주소 | 문제에서 "시작 주소"로 줌 |
| i | 행 인덱스 (0부터 시작) | 찾으려는 원소의 행 번호 |
| j | 열 인덱스 (0부터 시작) | 찾으려는 원소의 열 번호 |
| **nⱼ** | **열의 총 개수** | ⚠️ 행 개수(nᵢ)랑 헷갈리지 말 것! |
| ℓ (l) | 원소 하나의 크기 (byte) | int=4, short=2, double=8, char=1 |

### 열 우선 순서 (참고용, 시험에 가끔 나옴)

$$A[i][j]의\ 위치 = \alpha + (j \times n_i + i) \times \ell$$

> nᵢ = 행의 총 개수

---

## 3. 풀이 4단계 (이 순서로 항상 풀기)

```
1단계: 주어진 값 정리  →  α, ni, nj, i, j, ℓ 추출
2단계: 공식에 대입      →  α + (i × nj + j) × ℓ
3단계: 괄호 먼저 계산  →  (i × nj + j) 를 먼저
4단계: 곱한 뒤 α 더함  →  최종 주소 도출
```

---

## 4. 예제 풀이 (슬라이드 원본)

### 문제 (교재 126p 그대로)

> `int sale[2][4]` 배열의 시작 주소가 **13474812** 일 때,  
> `sale[1][2]` 의 메모리 주소를 구하시오.

### 풀이

**1단계: 값 정리**
```
α  = 13474812   ← 시작 주소
ni = 2           ← 행 개수 (2행)
nj = 4           ← 열 개수 (4열)  ← 공식에 쓰는 건 이거!
i  = 1           ← 찾는 원소의 행
j  = 2           ← 찾는 원소의 열
ℓ  = 4           ← int = 4byte
```

**2단계: 공식 대입**
```
위치 = α + (i × nj + j) × ℓ
     = 13474812 + (1 × 4 + 2) × 4
```

**3단계: 괄호 먼저**
```
     = 13474812 + (4 + 2) × 4
     = 13474812 + 6 × 4
     = 13474812 + 24
```

**4단계: 최종 답**
```
     = 13474836
```

> ✅ **답: 13474836**

## 5. 유형별 풀이 완전 정리

### 유형 1 — 기본형 (위치 구하기)

> 가장 자주 나오는 유형. 위의 4단계 그대로 적용.

**예제:** `int arr[3][5]`, 시작주소 α=2000000 일 때 `arr[2][3]` 의 주소는?

```
α=2000000, ni=3, nj=5, i=2, j=3, ℓ=4

위치 = 2000000 + (2 × 5 + 3) × 4
     = 2000000 + 13 × 4
     = 2000000 + 52
     = 2000052
```

---

### 유형 2 — 자료형이 다른 경우 (ℓ 주의)

> short, double, char 등 ℓ 값이 달라지는 경우

**자료형별 ℓ 정리:**
| 자료형 | 크기 |
|--------|------|
| char   | 1 byte |
| short  | 2 byte |
| int    | 4 byte |
| double | 8 byte |

**예제:** `double dbl[5][3]`, 시작주소 α=8000000 일 때 `dbl[3][1]` 의 주소는?

```
α=8000000, nj=3, i=3, j=1, ℓ=8

위치 = 8000000 + (3 × 3 + 1) × 8
     = 8000000 + 10 × 8
     = 8000000 + 80
     = 8000080
```

---

### 유형 3 — 역방향 (인덱스 거꾸로 구하기)

> 주소가 주어지고 i, j 를 구하는 유형

**예제:** `int arr[4][5]`, 시작주소 α=1000000,  
어떤 원소의 주소가 **1000060** 이다. 이 원소는 `arr[i][j]` 에서 i, j 는?

**풀이:**
```
1000060 = 1000000 + (i × 5 + j) × 4

(i × 5 + j) × 4 = 60
 i × 5 + j       = 15

→ i=2, j=5? → j는 열 개수(5)보다 작아야 함 → 불가
→ i=3, j=0? → 3×5+0 = 15 ✓  → arr[3][0]
```

**역방향 풀이 공식:**
```
오프셋  = (주소 - α) ÷ ℓ
i      = 오프셋 ÷ nj  (몫)
j      = 오프셋 % nj  (나머지)
```

**위 예제 재풀이:**
```
오프셋 = (1000060 - 1000000) ÷ 4 = 60 ÷ 4 = 15
i      = 15 ÷ 5 = 3  (몫)
j      = 15 % 5 = 0  (나머지)
→ arr[3][0]
```

---

### 유형 4 — 행 우선 vs 열 우선 비교

> "C 언어는 어떤 방식을 사용하는가?" 또는 둘 다 계산하라는 유형

| 구분 | 공식 | 저장 순서 |
|------|------|----------|
| 행 우선 (C 사용) | α + (i×**nⱼ** + j) × ℓ | 0행 전체 → 1행 전체 → ... |
| 열 우선 (Fortran 사용) | α + (j×**nᵢ** + i) × ℓ | 0열 전체 → 1열 전체 → ... |

**예제:** `int sale[2][4]`, α=13474812, `sale[1][2]` 의 주소를 두 방식으로 구하면?

```
행 우선: 13474812 + (1×4 + 2) × 4 = 13474812 + 24 = 13474836
열 우선: 13474812 + (2×2 + 1) × 4 = 13474812 + 20 = 13474832
```

> ✅ **C 컴파일러는 항상 행 우선 순서** 사용

---

### 유형 5 — 저장 순서 나열하기

> 배열 원소들이 메모리에 저장되는 순서를 쓰라는 유형

**예제:** `int sale[2][4]` 의 메모리 저장 순서는?

**행 우선:**
```
sale[0][0] → sale[0][1] → sale[0][2] → sale[0][3]
           → sale[1][0] → sale[1][1] → sale[1][2] → sale[1][3]
```

**열 우선:**
```
sale[0][0] → sale[1][0]
           → sale[0][1] → sale[1][1]
           → sale[0][2] → sale[1][2]
           → sale[0][3] → sale[1][3]
```

---

### 유형 6 — 배열 크기·범위 확인 문제

> 특정 주소가 배열 범위 안에 있는지 확인

**예제:** `int arr[3][4]`, α=5000 일 때 주소 5048은 어떤 원소인가?

```
오프셋 = (5048 - 5000) ÷ 4 = 48 ÷ 4 = 12
i = 12 ÷ 4 = 3  → ⚠️ 행 인덱스 3은 arr[3][4]에서 범위 초과! (0~2만 유효)
→ 배열 범위 밖 (유효하지 않은 접근)
```

## 6. 자주 하는 실수 TOP 5

| 실수 | 틀린 예 | 올바른 예 |
|------|---------|----------|
| nᵢ 와 nⱼ 혼동 | (i × **ni** + j) × ℓ | (i × **nj** + j) × ℓ |
| 인덱스를 1부터 시작 | i=1이면 첫 번째 행 | i=0이 첫 번째 행 |
| ℓ 빠뜨림 | α + (i×nj + j) | α + (i×nj + j) **× ℓ** |
| 열 우선 공식을 행 우선에 사용 | (j×ni + i) × ℓ | (i×nj + j) × ℓ |
| α 더하는 것을 잊음 | (i×nj + j) × ℓ | **α** + (i×nj + j) × ℓ |


## 8. 최종 체크리스트 (시험장에서)

- [ ] α(시작주소), nj(열 개수), i, j, ℓ 값을 먼저 정리했는가?
- [ ] nⱼ 는 **열의 개수** (행 개수 nᵢ 아님!)
- [ ] 인덱스는 **0부터** 시작
- [ ] C언어 → **행 우선** 공식 사용
- [ ] 괄호 **먼저** 계산 후 ℓ 곱하기
- [ ] 마지막에 α **더하기**
