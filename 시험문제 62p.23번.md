# 시간 복잡도 (Time Complexity) 정리

## 1. 시간 복잡도란?

알고리즘이 실행될 때 **입력 크기(n)에 따라 얼마나 많은 연산을 수행하는지** 나타내는 척도.
Big-O 표기법(O)으로 나타내며, **최악의 경우**를 기준으로 한다.

---

## 2. 성능 순서 (빠름 → 느림)

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

| 표기법 | 이름 | 설명 | 예시 |
|---|---|---|---|
| O(1) | 상수 시간 | 입력 크기 상관없이 항상 일정 | 배열 인덱스 접근 |
| O(log n) | 로그 시간 | 입력이 2배 증가해도 연산은 1번 증가 | 이진 탐색 |
| O(n) | 선형 시간 | 입력 크기에 비례 | 선형 탐색 |
| O(n log n) | 선형 로그 시간 | 정렬 알고리즘의 이상적인 복잡도 | 병합 정렬, 퀵 정렬 |
| O(n²) | 이차 시간 | 이중 반복문 | 버블 정렬, 삽입 정렬 |
| O(2ⁿ) | 지수 시간 | 입력 1 증가 시 연산이 2배 | 피보나치(재귀), 부분집합 |
| O(n!) | 팩토리얼 시간 | 가장 느림 | 순열 탐색 |

---

## 3. 시험 문제 유형

### 문제 예시
> 알고리즘 A, B, C, D의 시간 복잡도가 다음과 같을 때 성능이 좋은 것부터 순서대로 나열하시오.
> - A: O(n²)
> - B: O(1)
> - C: O(n log n)
> - D: O(2ⁿ)

### 풀이
성능이 좋다 = 실행 시간이 짧다 = Big-O 값이 작다

```
B → C → A → D
O(1) < O(n log n) < O(n²) < O(2ⁿ)
```

---

## 4. 각 복잡도 상세 설명

### O(1) - 상수 시간
```python
arr = [1, 2, 3, 4, 5]
print(arr[0])  # 배열 크기 상관없이 항상 1번 연산
```
- 입력 크기가 아무리 커도 연산 횟수 **고정**
- 가장 이상적인 복잡도

---

### O(log n) - 로그 시간
```python
# 이진 탐색
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
```
- n=1000일 때 약 10번만 연산
- 매 단계마다 탐색 범위를 **절반**으로 줄임

---

### O(n) - 선형 시간
```python
# 선형 탐색
def linear_search(arr, target):
    for i in arr:      # n번 반복
        if i == target:
            return True
```
- 입력이 2배 → 연산도 2배
- 반복문 1개

---

### O(n log n) - 선형 로그 시간
```python
arr = [5, 3, 1, 4, 2]
arr.sort()  # 파이썬 기본 정렬 = O(n log n)
```
- 정렬 알고리즘의 **이론적 최솟값**
- 병합 정렬, 힙 정렬, 퀵 정렬(평균)

---

### O(n²) - 이차 시간
```python
# 버블 정렬
def bubble_sort(arr):
    for i in range(len(arr)):       # n번
        for j in range(len(arr)):   # n번
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
```
- **이중 반복문** → n × n = n²
- n=1000이면 100만 번 연산

---

### O(2ⁿ) - 지수 시간
```python
# 피보나치 재귀
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)  # 매 호출마다 2개로 분기
```
- n이 조금만 커져도 연산이 **폭발적으로 증가**
- n=30이면 약 10억 번 연산

---

## 5. n 크기별 실행 시간 비교

| n | O(1) | O(log n) | O(n) | O(n log n) | O(n²) | O(2ⁿ) |
|---|---|---|---|---|---|---|
| 10 | 1 | 3 | 10 | 30 | 100 | 1,024 |
| 100 | 1 | 7 | 100 | 664 | 10,000 | 엄청남 |
| 1,000 | 1 | 10 | 1,000 | 10,000 | 1,000,000 | 불가능 |

---

## 6. 핵심 요약

- **O(1)**: 최고, 항상 일정
- **O(log n)**: 매우 좋음, 이진 탐색
- **O(n)**: 보통, 반복문 1개
- **O(n log n)**: 괜찮음, 효율적인 정렬
- **O(n²)**: 나쁨, 이중 반복문
- **O(2ⁿ)**: 최악, 지수적 증가
