### Q01. 다음 중 스택에 대한 옳은 내용으로만 나열된 것은?

```
ㄱ. FIFO 방식으로 처리된다.
ㄴ. 순서 리스트의 뒤에서 노드가 삽입되며 앞에서 노드가 제거된다.
ㄷ. 선형 리스트의 양쪽 끝에서 삽입과 삭제가 모두 가능한 자료구조이다.
ㄹ. 인터럽트 처리, 서브루틴 작업 등에 응용된다.

① ㄱ, ㄴ    ② ㄴ, ㄷ    ③ ㄹ    ④ ㄱ, ㄴ, ㄷ, ㄹ
```

> **✅ 정답: ③ ㄹ**
>
> - ㄱ. ❌ 스택은 **LIFO**(후입선출). FIFO는 큐(Queue).
> - ㄴ. ❌ 뒤 삽입, 앞 제거 = **큐**의 설명.
> - ㄷ. ❌ 양쪽 삽입·삭제 = **데크(Deque)**의 설명.
> - ㄹ. ✅ 스택 응용: 인터럽트 처리, 서브루틴 호출, 수식 계산 등.

> **✅ 정답: ② data**
>

### Q09. 스택 알고리즘에서 T가 스택 포인터이고, m이 스택 길이일 때, 서브루틴 'AA'가 처리해야 하는 것은?

```
T ← T + 1
if T > m then goto AA
else STACK(T) ← item

① 오버플로 처리    ② 언더플로 처리    ③ 삭제 처리    ④ 삽입 처리

> **✅ 정답: ① 오버플로 처리**

> - AA 서브루틴은 오버플로 예외 처리를 담당

### Q13. 순서가 A, B, C, D로 정해진 입력 자료를 스택에 입력하였다가 출력한 결과로 가능한 것은?

```
① D, B, C, A    ② D, C, A, B    ③ C, D, A, B    ④ B, C, D, A
```

> **✅ 정답: ④ B, C, D, A**
>
> ```
> push A → push B → pop(B) → push C → pop(C) → push D → pop(D) → pop(A)
> → 출력: B, C, D, A  ✅
> ```
> - ①: D를 먼저 꺼내면 이후 스택은 C,B,A 역순 → D,C,B,A만 가능. D,B,C,A는 불가
> - ②③: LIFO 규칙 위반 — 불가능한 순서

### Q17. stack[0:99] 배열, top 초깃값 -1일 때 push 함수의 ㉠과 pop 함수의 ㉡은?

```c
void push(int * top, int data) {
  if (*top >= 99) { stack_full(); return; }
  ㉠;   // top 증가 후 데이터 저장
}

int pop(int * top) {
  if (*top == -1) { return stack_empty(); }
  ㉡;   // 현재 top 값 반환 후 top 감소
}

①  ㉠ stack[*top++]=data     ㉡ return stack[--*top]
②  ㉠ stack[*top++]=data     ㉡ return stack[*top--]
③  ㉠ stack[++*top]=data     ㉡ return stack[*top--]
④  ㉠ stack[++*top]=data     ㉡ return stack[--*top]
```

> **✅ 정답: ④ ㉠ `stack[++*top]=data`  /  ㉡ `return stack[--*top]`**
>
> - **push**: top이 -1에서 시작하므로 먼저 증가(`++*top`) 후 저장 → `stack[++*top]=data`
> - **pop**: top 먼저 감소(`--*top`) 후 해당 위치 반환 → `return stack[--*top]`

### Q18. 연결 리스트 스택 그림에 대한 설명으로 옳지 않은 것은?

```
(ㄱ) push 연산 수행 전:  top → b → c → d
(ㄴ) push 연산 수행 후:  top → a → b → c → d

① 스택에 가장 최근에 입력된 자료는 top이 지시한다.
② 스택에 입력된 자료 중 d가 가장 오래된 자료이다.
③ (ㄴ)에서 자료 c를 가져오려면 pop 연산이 2회 필요하다.
④ (ㄱ)에서 자료의 입력된 순서는 d, c, b이다.
```

> **✅ 정답: ③**
>
> - (ㄴ) `top→a→b→c→d` 상태에서 c를 꺼내려면:
>   - pop(a출력) → pop(b출력) → pop(c출력) = **총 3회 필요**
> - 2회가 아니라 3회이므로 ③이 틀린 설명
> - ① ✅ ② ✅ (d가 가장 먼저 삽입) ④ ✅ (d,c,b 순서 push)

### Q19. 연결 리스트 스택의 삭제 알고리즘에서 ㉠~㉢에 들어갈 문장은?

```c
NODE *top = NULL;   // 스택 최상위 노드를 가리키는 포인터

int pop(void) {
  NODE *temp;  int ret;
  if (㉠) { printf("Stack underflow."); return -1; }
  ret = top->key;
  ㉡;
  ㉢;
  free(temp);
  return ret;
}

①  ㉠ top->next==NULL   ㉡ temp=top->next   ㉢ top=top->next
②  ㉠ top->next==NULL   ㉡ temp=top         ㉢ top->next=temp->next
③  ㉠ top==NULL         ㉡ temp=top         ㉢ top=top->next
④  ㉠ top==NULL         ㉡ temp=top->next   ㉢ top->next=temp->next
```

> **✅ 정답: ③**
>
> - ㉠ `top==NULL`: 스택이 비었는지 확인 (NULL이면 언더플로)
> - ㉡ `temp=top`: 삭제할 현재 top 노드를 temp에 저장
> - ㉢ `top=top->next`: top 포인터를 다음 노드로 이동
> - 이후 `free(temp)`로 메모리 해제 후 ret 반환

### Q01. 자료구조에 대한 설명으로 옳지 않은 것은?

> **✅ 정답: ④ 큐는 비선형 구조에 해당한다.**

> - 큐(Queue)는 **선형 자료구조(Linear Data Structure)**

### Q02. 다음 중 큐에 대한 설명으로 옳은 것은?

```
① 큐의 크기는 항상 미리 정해져 있어야 한다.
② 자료의 삽입과 삭제는 같은 위치인 끝에서 일어난다.
③ 자료의 입출력은 LIFO(Last-In-First-Out) 순서로 일어난다.

> - ① ❌ 연결 리스트로 구현하면 크기를 미리 정하지 않아도 됨
> - ② ❌ 삽입은 rear(뒤), 삭제는 front(앞) — 서로 다른 위치
> - ③ ❌ LIFO는 스택. 큐는 **FIFO(First-In-First-Out)**
> - ④ ✅ 삽입(enqueue): rear에 추가 O(1), 삭제(dequeue): front에서 제거 O(1)

### Q03. 큐에 대한 설명으로 틀린 것은?

```

> **✅ 정답: ① 자료의 삽입과 삭제가 Top에서 이루어진다.**
>

> - 큐: 삽입(enqueue) → **Rear(뒤)**, 삭제(dequeue) → **Front(앞)**

>
> 선형 큐에서 삭제가 일어나면 front가 뒤로 이동하지만 앞쪽 빈 공간은 재사용되지 않음.
> rear가 배열 끝에 도달하면 앞에 빈 공간이 있어도 포화 상태로 판단 → 공간 낭비.

### Q17. 연결 리스트 포인터를 하나만 사용해 큐를 구현할 때 가장 효율적인 구조는?

```
① ptr이 첫 노드를 가리키는 원형 리스트:     A→B→C→D→(A로 순환)
② ptr이 마지막 노드를 가리키는 원형 리스트:  A→B→C→D→(A로 순환), ptr→D
③ ptr이 첫 노드를 가리키는 역원형 리스트:    D→C→B→A→(D로 순환)
④ ptr이 마지막 노드를 가리키는 역원형 리스트: D→C→B→A→(D로 순환), ptr→A

### Q18. 원형 큐에서 포화 상태와 공백 상태의 조건을 설명하시오.

> **[공백 상태 (Empty)]**
> ```
> front == rear
> ```
> front와 rear가 같은 위치 → 큐에 데이터 없음
>
> **[포화 상태 (Full)]**
> ```
> front == (rear + 1) % SIZE
> ```
> rear의 다음 위치가 front와 같음 → 더 이상 삽입 불가
> (원형 큐는 배열 크기 n일 때 실제로 **n-1개** 원소만 저장 가능)
>
> **[공식 정리]**
> ```
> 삽입(enqueue): rear = (rear + 1) % SIZE  후 데이터 저장
> 삭제(dequeue): front = (front + 1) % SIZE  후 데이터 반환
> ```
