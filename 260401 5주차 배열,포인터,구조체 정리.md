## 1. 배열 (Array)

### 개념

- 같은 자료형의 데이터를 메모리에 **연속으로** 저장하는 그룹
- **인덱스(index)** 로 각 요소를 구별하며, 인덱스는 항상 **0부터 시작**
- 모든 자료형에 대해 배열로 구성 가능 (1차원, 2차원, 3차원...)

### 1차원 배열

#### 선언 및 초기화

```c
int score[4];                     // int형 4개짜리 배열 선언
int score[4] = {10, 20, 30, 40};  // 선언과 동시에 초기화
char name[] = "Hello";            // 문자 배열 (끝에 \0 자동 추가)
```

#### 메모리 구조 (`int score[4]`)

| score[0] | score[1] | score[2] | score[3] |
|---|---|---|---|
| 10 (주소: 100) | 20 (주소: 104) | 30 (주소: 108) | 40 (주소: 112) |

> `int`는 4바이트 → 인접 요소의 주소가 4씩 증가

#### 문자 배열

```c
char s1[] = "String";                        // 문자열로 초기화
char s2[] = {'S','t','r','i','n','g','\0'};  // 문자 리스트로 초기화
// 반드시 마지막에 \0 (널 문자) 포함 필요
```

---

### 다차원 배열

#### 선언 형식

```c
int arr[2][3];           // 2행 3열 2차원 배열
int arr[2][3][4];        // 3차원 배열
```

#### 논리 구조 vs 물리 구조

`int arr[2][3] = {{1,2,3},{4,5,6}};`

**논리적 모습 (표)**

|  | 열 0 | 열 1 | 열 2 |
|---|---|---|---|
| **행 0** | 1 | 2 | 3 |
| **행 1** | 4 | 5 | 6 |

**실제 메모리 (1줄로 연속 저장)**

| `[0][0]` | `[0][1]` | `[0][2]` | `[1][0]` | `[1][1]` | `[1][2]` |
|---|---|---|---|---|---|
| 1 | 2 | 3 | 4 | 5 | 6 |

> 행 우선(row-major) 방식으로 저장 — 0행 전체 → 1행 전체 순서

#### 다차원 배열 초기화

```c
// 2차원 배열 초기화
int arr[2][3] = {
    {1, 2, 3},   // 0행
    {4, 5, 6}    // 1행
};

// 1줄로도 가능 (동일한 결과)
int arr[2][3] = {1, 2, 3, 4, 5, 6};
```

#### 문자 다차원 배열

```c
char names[3][10] = {"Kim", "Lee", "Park"};
// names[0] → "Kim"
// names[1] → "Lee"
// names[2] → "Park"
```

## 2. 포인터 (Pointer)

### 개념

- 변수의 **메모리 주소값**을 저장하는 특별한 변수
- 포인터가 어떤 변수의 주소를 갖고 있으면 **"그 변수를 가리킨다"** 고 표현
- 포인터를 통해 연결된 주소의 변수 영역을 액세스 가능

### 핵심 연산자 2개

| 연산자 | 이름 | 설명 |
|---|---|---|
| `&` | 주소 연산자 | 변수의 메모리 주소를 가져옴. 예) `&a` → a의 주소 |
| `*` | 참조 연산자 | 포인터가 가리키는 주소의 값을 읽거나 씀. 예) `*p` → p가 가리키는 값 |

### 선언 및 사용

```c
int a = 10;
int *p;        // 포인터 선언 (int형 주소를 저장)
p = &a;        // p에 a의 주소 저장

printf("%d", *p);  // 10 출력 → p가 가리키는 값
*p = 20;           // a의 값이 20으로 변경됨
```

#### 메모리 시각화

```
변수 a                    포인터 p
┌─────────────┐          ┌─────────────┐
│ 값:  10     │  ←───── │ 값:  200    │  (a의 주소)
│ 주소: 200   │          │ 주소: 300   │
└─────────────┘          └─────────────┘

*p  → p가 가리키는 주소(200)의 값 → 10
&a  → a의 주소 → 200
```

### 포인터 초기화 방법 5가지

```c
int a = 10;
int arr[3] = {1, 2, 3};
int *p;

// ➊ 주소 연산자로 변수 주소 지정
p = &a;

// ➋ 동적 메모리 할당
p = (int*)malloc(sizeof(int));

// ➌ 문자형 포인터에 문자열 주소
char *str = "Hello";

// ➍ 배열 이름 = 배열 시작 주소
p = arr;         // arr == &arr[0]

// ➎ 배열 첫 요소의 주소
p = &arr[0];     // ➍와 동일한 결과
```

### 포인터 배열

```c
char *names[3] = {"Kim", "Lee", "Park"};
// names[0] → "Kim" 문자열의 시작 주소
// names[1] → "Lee" 문자열의 시작 주소
// names[2] → "Park" 문자열의 시작 주소

// 2차원 문자 배열보다 메모리 효율적
```

#### 2차원 배열 vs 포인터 배열 비교

| 구분 | 2차원 문자 배열 | 포인터 배열 |
|---|---|---|
| 선언 | `char names[3][10]` | `char *names[3]` |
| 메모리 | 고정 크기 (3×10=30바이트) | 문자열 길이만큼만 사용 |
| 특징 | 각 행 크기 동일 | 각 문자열 길이 달라도 됨 |

### 포인터의 포인터 (이중 포인터)

```c
int a = 10;
int *p = &a;
int **pp = &p;  // pp → p → a

printf("%d", **pp);  // 10 출력
// **pp → *(*pp) → *(p) → a의 값
```

## 3. 구조체 (Structure)

### 개념

- **서로 다른 자료형**을 그룹으로 묶어 하나의 새 자료형으로 정의
- 배열은 같은 자료형만 묶을 수 있지만, 구조체는 다른 자료형도 묶을 수 있음
- 레코드(여러 필드의 묶음)를 만들 때 유용

#### 배열 vs 구조체

| 구분 | 배열 | 구조체 |
|---|---|---|
| 자료형 | 동일한 자료형만 가능 | 서로 다른 자료형 가능 |
| 접근 방법 | 인덱스 `arr[i]` | 점/화살표 연산자 |
| 용도 | 동종 데이터 목록 | 복합 레코드 정의 |

### 사용 3단계

```c
// ➊ 구조체형 선언 (틀 정의)
struct Student {
    char  name[20];
    int   age;
    float score;
};

// ➋ 구조체 변수 선언
struct Student s1;

// ➌ 사용 (점 연산자로 접근)
s1.age = 20;
s1.score = 95.5;
```

#### typedef로 간단하게 사용하기

```c
typedef struct {
    char  name[20];
    int   age;
    float score;
} Student;

Student s1;   // struct 키워드 없이 사용 가능
```

### 데이터 항목 접근 연산자

| 연산자 | 사용 조건 | 예시 |
|---|---|---|
| `.` (점) | 일반 구조체 변수 | `s1.age = 20;` |
| `->` (화살표) | 구조체 포인터 변수 | `p->age = 20;` |

```c
struct Student s1;
struct Student *p = &s1;

// 두 방법은 완전히 동일
p->age = 21;      // 포인터로 접근 (권장)
(*p).age = 21;    // 동일 — 괄호 필수! *p.age 는 오류
```

> ⚠️ `*p.age` 처럼 괄호 없이 쓰면 오류 — 반드시 `(*p).age` 또는 `p->age` 사용

### 구조체 초기화

```c
struct Student s1 = {"Kim", 20, 95.5};
// 선언 순서(name → age → score)에 맞춰 초기값 지정
```

### 구조체 가능한 연산

```c
struct Student s1 = {"Kim", 20, 95.5};
struct Student s2;

// 항목 참조
s1.age = 21;

// 변수 복사 (통째로 한 번에 가능)
s2 = s1;   // s1의 모든 내용이 s2로 복사

// 주소 구하기
struct Student *p = &s1;
```

## 4. 핵심 요약

| 주제 | 핵심 개념 | 주요 문법 |
|---|---|---|
| 배열 | 같은 자료형 연속 저장, 인덱스 0 시작 | `int arr[4];` `arr[0];` |
| 포인터 | 주소를 저장하는 변수 | `int *p;` `&a;` `*p;` |
| 구조체 | 다른 자료형을 묶는 복합 자료형 | `struct` `.`(점) `->`(화살표) |

### 포인터 연산자 한눈에 보기

```
변수 a → &a → 주소값
주소값 → *p → 변수 a의 값

int a = 10;
int *p = &a;

&a  == 200  (a가 저장된 주소)
p   == 200  (p에 저장된 값 = a의 주소)
*p  == 10   (p가 가리키는 값 = a의 값)
```

### 구조체 연산자 한눈에 보기

```
일반 변수  →  점(.) 연산자    →  s1.age
포인터     →  화살표(->) 연산자  →  p->age  (= (*p).age)
```
