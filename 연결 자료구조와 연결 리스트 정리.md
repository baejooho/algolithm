## 1. 연결 자료구조와 연결 리스트의 이해

### 1-1. 순차 자료구조의 문제점

- 삽입·삭제 후 연속적인 물리 주소를 유지하기 위해 **원소 이동 작업**과 시간이 소요됨
- 원소의 개수가 많고 삽입·삭제가 빈번할 경우 **성능 저하** 발생
- 배열 기반 구현으로 **메모리 사용 비효율** 문제를 그대로 가짐
- → 연산 시간과 저장 공간 문제를 개선한 자료 표현 방법이 필요

### 1-2. 연결 자료구조 (Linked Data Structure)

| 항목 | 내용 |
|------|------|
| 논리적/물리적 순서 | 불일치 (논리 순서는 포인터로 유지) |
| 오버헤드 | 물리적 순서를 맞출 필요 없으므로 오버헤드 없음 |
| 메모리 | 여러 작은 공간을 연결 → 크기 변경 유연, 효율적 |
| 연결 방식 | 각 원소에 저장된 **다음 원소의 주소**로 순서 연결 |

#### 연결 리스트 종류

- 단순 연결 리스트 (Singly Linked List)
- 원형 연결 리스트 (Circular Linked List)
- 이중 연결 리스트 (Doubly Linked List)
- 이중 원형 연결 리스트

### 1-3. 연결 리스트의 노드

노드 = **`<원소, 주소>`** 구조

```
┌───────────┬───────────┐
│ data field│ link field│
│  (원소 값) │ (다음 주소)│
└───────────┴───────────┘
```

| 필드 | 설명 |
|------|------|
| **데이터 필드** (data field) | 원소의 값 저장. 원소 형태에 따라 하나 이상의 필드로 구성 |
| **링크 필드** (link field) | 다음 노드의 주소 저장. 포인터 변수 사용 |

### 1-4. 순차 자료구조 vs 연결 자료구조 비교

| 비교 항목 | 순차 자료구조 | 연결 자료구조 |
|-----------|--------------|--------------|
| 물리적 순서 | 논리 순서와 일치 | 논리 순서와 불일치 |
| 삽입·삭제 | O(n) — 원소 이동 필요 | O(1) — 포인터만 변경 |
| 메모리 | 연속 공간 필요 | 불연속 공간 사용 가능 |
| 탐색 | O(1) — 인덱스 직접 접근 | O(n) — 순차 탐색 |
| 크기 변경 | 어려움 (배열 크기 고정) | 유연함 |

### 1-5. 연결 리스트 기본 개념

```
week → [월|100] → [화|200] → [수|300] → ... → [일|NULL]
 ↑
시작 포인터
```

- `week` : 연결 리스트의 **시작(첫 번째 노드)을 가리키는 포인터 변수**
- 마지막 노드의 링크 필드 : **NULL** 저장 (리스트 끝 표시)
- **공백 연결 리스트** : 포인터 변수 week에 NULL 저장
- 노드 필드 접근 : **점 연산자(.)** 사용
  - `week.data` : 첫 번째 노드의 데이터 필드 값
  - `week.link` : 첫 번째 노드의 링크 필드 값 (다음 노드 주소)

#### C 구조체 정의 예시

```c
struct ListNode {
    char data[10];         // 데이터 필드
    struct ListNode *link; // 링크 필드 (다음 노드 포인터)
};
```

## 2. 단순 연결 리스트

### 2-1. 개념

- 노드가 **하나의 링크 필드**에 의해 다음 노드와 연결
- 별칭: 선형 연결 리스트, 단순 연결 선형 리스트

```
L → [A|●]──→[B|●]──→[C|NULL]
```

### 2-2. 삽입 연산

#### 첫 번째 노드로 삽입

```
삽입 전:  L ──→ [A|●]──→[B|●]──→[C|NULL]
삽입 후:  L ──→ [X|●]──→[A|●]──→[B|●]──→[C|NULL]
```

**알고리즘**
```
① new ← getNode()         // 새 노드 할당
② new.data ← x            // 데이터 저장
③ new.link ← L            // new가 현재 첫 번째 노드를 가리킴
④ L ← new                 // L이 new를 첫 번째 노드로 가리킴
```

#### 중간 노드로 삽입 (pre 다음에 삽입)

```
삽입 전:  L ──→ [A|●]──→[B|●]──→[C|NULL]
                  pre
삽입 후:  L ──→ [A|●]──→[X|●]──→[B|●]──→[C|NULL]
```

**알고리즘 (공백 리스트가 아닌 경우)**
```
① new ← getNode()
② new.data ← x
③-a new.link ← pre.link   // new가 pre의 다음 노드를 가리킴
③-b pre.link ← new        // pre가 new를 가리킴
```

> ⚠️ **순서 주의**: ③-a 를 반드시 먼저 실행해야 다음 노드 주소를 잃지 않음

#### 마지막 노드로 삽입

```
삽입 전:  L ──→ [A|●]──→[B|●]──→[C|NULL]
삽입 후:  L ──→ [A|●]──→[B|●]──→[C|●]──→[X|NULL]
```

**알고리즘 (공백 리스트가 아닌 경우)**
```
① new ← getNode()
② new.data ← x
③ new.link ← NULL
④-a temp ← L                        // temp를 첫 번째 노드에서 시작
④-b while (temp.link ≠ NULL)        // 마지막 노드까지 순회
       temp ← temp.link
④-c temp.link ← new                 // 마지막 노드가 new를 가리킴
```

### 2-3. 삭제 연산

```
삭제 전:  L ──→ [A|●]──→[B|●]──→[C|●]──→[D|NULL]
                  pre      old
삭제 후:  L ──→ [A|●]──────────→[C|●]──→[D|NULL]
```

**알고리즘 (pre 다음 노드 old 삭제)**
```
① old ← pre.link             // 삭제할 노드 지정
② if (old = NULL) then return // 삭제할 노드 없으면 반환
③ pre.link ← old.link        // pre가 old의 다음 노드를 가리킴
④ returnNode(old)             // old 노드 메모리 반환
```

### 2-4. 탐색 연산

```
① temp ← L
② while (temp ≠ NULL) {
     if (temp.data = x) then return temp   // 탐색 성공
     else temp ← temp.link                 // 다음 노드로 이동
   }
③ return temp   // 탐색 실패 (NULL 반환)
```

### 2-5. 역순 변환 (Q08 관련)

세 개의 포인터(p, q, r)를 사용하여 연결 순서를 뒤집음

```c
p = start;   // 현재 처리할 노드
q = NULL;    // 역순으로 구성된 리스트의 head
while (p != NULL) {
    r = q;           // ㉠ 이전 노드 백업
    q = p;           //    현재 노드를 역순 리스트 head로
    p = p->link;     // p를 다음 노드로 이동
    q->link = r;     // 현재 노드의 link를 이전 노드로 연결
}
start = q;
```

**예시: A → B → C → NULL**

| 단계 | p | q | r | 결과 |
|------|---|---|---|------|
| 초기 | A | NULL | - | - |
| 1회 | B | A | NULL | A→NULL |
| 2회 | C | B | A | B→A→NULL |
| 3회 | NULL | C | B | C→B→A→NULL |

→ 최종: `start = C`, 리스트: `C → B → A → NULL`

## 3. 원형 연결 리스트

### 3-1. 개념

- 단순 연결 리스트에서 **마지막 노드가 첫 번째 노드를 가리키도록** 구성
- 링크를 따라 계속 순회하면 이전 노드 접근 가능

```
단순:   L → [A|●]──→[B|●]──→[C|NULL]

원형:   L → [A|●]──→[B|●]──→[C|●]
             ↑________________________|
```

### 3-2. 삽입 연산

#### 마지막 노드로 삽입

**공백 리스트인 경우**
```
① CL ← new
② new.link ← new    // 자기 자신을 가리킴 (원형 유지)
```

**공백 리스트가 아닌 경우**
```
① temp ← CL
② while (temp.link ≠ CL)    // 마지막 노드까지 순회
     temp ← temp.link
③ new.link ← temp.link      // new가 첫 번째 노드를 가리킴
④ temp.link ← new           // 마지막 노드가 new를 가리킴
⑤ CL ← new                  // CL을 새 노드(새 첫 번째)로 갱신
```

#### 중간 노드로 삽입 (pre 다음에 삽입)

```
① new.link ← pre.link   // new가 pre의 다음을 가리킴
② pre.link ← new        // pre가 new를 가리킴
```

### 3-3. 삭제 연산

```
① old ← pre.link           // 삭제할 노드 지정
② pre.link ← old.link      // pre가 old의 다음을 가리킴
③ if (old = CL) then        // 삭제 노드가 첫 번째였다면
     CL ← old.link          //   두 번째 노드를 첫 번째로 갱신
④ returnNode(old)
```

## 4. 이중 연결 리스트

### 4-1. 개념

- **양쪽 방향**으로 순회할 수 있도록 노드를 연결한 리스트
- 노드 구조: `llink | data | rlink`

```
┌────────┬───────────┬────────┐
│ llink  │   data    │ rlink  │
│(왼쪽↩) │           │(오른쪽→)│
└────────┴───────────┴────────┘
```

| 필드 | 설명 |
|------|------|
| **llink** (left link) | 왼쪽(이전) 노드 주소 |
| **rlink** (right link) | 오른쪽(다음) 노드 주소 |

```
NULL ←[llink|A|rlink]⟷[llink|B|rlink]⟷[llink|C|rlink]→ NULL
```

- **원형 이중 연결 리스트**: 이중 연결 리스트를 원형으로 구성

### 4-2. 삽입 연산 (pre 다음에 new 삽입)

```
삽입 전:  ⟷ [pre] ⟷ [next] ⟷
삽입 후:  ⟷ [pre] ⟷ [new] ⟷ [next] ⟷
```

**알고리즘 (4단계 포인터 조정)**
```
① new.rlink ← pre.rlink     // new의 오른쪽 = pre의 오른쪽(기존 next)
② pre.rlink ← new           // pre의 오른쪽 = new
③ new.llink ← pre           // new의 왼쪽 = pre
④ new.rlink.llink ← new     // next의 왼쪽 = new
```

> ⚠️ **순서 주의**: ①을 먼저 실행해야 next 주소를 잃지 않음

### 4-3. 삭제 연산 (old 노드 삭제)

```
삭제 전:  ⟷ [lnode] ⟷ [old] ⟷ [rnode] ⟷
삭제 후:  ⟷ [lnode] ⟷ [rnode] ⟷
```

**알고리즘**
```
① old.llink.rlink ← old.rlink   // old 왼쪽 노드의 rlink = old의 rlink
② old.rlink.llink ← old.llink   // old 오른쪽 노드의 llink = old의 llink
③ returnNode(old)
```

## 5. 연결 리스트의 응용 및 구현

### 5-1. 다항식 표현

#### 노드 구조

```
┌────────┬──────┬────────┐
│  coef  │ expo │  link  │
│ (계수) │(지수)│(다음→) │
└────────┴──────┴────────┘
```

#### C 구조체 정의

```c
struct PolyNode {
    float coef;           // 계수
    int   expo;           // 지수
    struct PolyNode *link;
};
```

#### 예시

```
A(x) = 4x³ + 3x² + 5x
B(x) = 3x⁴ + x³ + 2x + 1

A: [4|3|●]──→[3|2|●]──→[5|1|NULL]
B: [3|4|●]──→[1|3|●]──→[2|1|●]──→[1|0|NULL]
```

### 5-2. 항 삽입 알고리즘 (appendTerm)

| 상황 | 처리 |
|------|------|
| 공백 리스트 | `PL ← new`, `last ← new` |
| 공백 아닌 경우 | `last.link ← new`, `last ← new` |

### 5-3. 다항식 덧셈 연산 C(x) = A(x) + B(x)

세 포인터 사용: `p` (A의 현재 항), `q` (B의 현재 항), `r` (C의 현재 항)

#### 경우별 처리

| 조건 | 처리 |
|------|------|
| `p.expo < q.expo` | B의 현재 항(q)을 C에 복사, q 이동 |
| `p.expo = q.expo` | 계수 합산: `r.coef = p.coef + q.coef`, p·q 모두 이동 |
| `p.expo > q.expo` | A의 현재 항(p)을 C에 복사, p 이동 |
| p 또는 q가 NULL | 남은 항을 C에 그대로 복사 |

#### 덧셈 알고리즘

```
① p ← A;  q ← B;  C ← 공백
② while (p ≠ NULL and q ≠ NULL) {
     if (p.expo < q.expo)      // ➋-b: B 항 복사
         appendTerm(C, q.coef, q.expo);  q ← q.link
     else if (p.expo = q.expo) // ➋-a: 계수 합산
         appendTerm(C, p.coef+q.coef, p.expo);  p ← p.link;  q ← q.link
     else                      // ➋-c: A 항 복사
         appendTerm(C, p.coef, p.expo);  p ← p.link
   }
③ while (p ≠ NULL)  // A 잔여 항 복사
     appendTerm(C, p.coef, p.expo);  p ← p.link
④ while (q ≠ NULL)  // B 잔여 항 복사
     appendTerm(C, q.coef, q.expo);  q ← q.link
⑤ return C
```

#### 예시: A(x) = 4x³+3x²+5x, B(x) = 3x⁴+x³+2x+1

| 단계 | p (A) | q (B) | 조건 | C에 추가 |
|------|--------|--------|------|----------|
| 1 | 4x³ | 3x⁴ | p.expo < q.expo | 3x⁴ |
| 2 | 4x³ | x³ | p.expo = q.expo | 5x³ |
| 3 | 3x² | 2x | p.expo > q.expo | 3x² |
| 4 | 5x | 2x | p.expo = q.expo | 7x |
| 5 | NULL | 1 | q 잔여 | 1 |

**결과: C(x) = 3x⁴ + 5x³ + 3x² + 7x + 1**

## 요약 비교표

| 연결 리스트 종류 | 링크 수 | 방향 | 마지막 노드 link |
|----------------|---------|------|-----------------|
| 단순 연결 리스트 | 1 | 단방향 | NULL |
| 원형 연결 리스트 | 1 | 단방향 (순환) | 첫 번째 노드 주소 |
| 이중 연결 리스트 | 2 (llink, rlink) | 양방향 | NULL |
| 이중 원형 연결 리스트 | 2 | 양방향 (순환) | 첫 번째 노드 주소 |

| 연산 | 단순 연결 리스트 | 배열(순차) |
|------|----------------|-----------|
| 삽입·삭제 (위치 알 때) | **O(1)** | O(n) |
| 탐색 | O(n) | **O(1)** |
| 메모리 | 포인터 추가 공간 필요 | 연속 메모리 |
